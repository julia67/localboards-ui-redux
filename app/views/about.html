Local Boards

LocalBoards.org organizes board openings, qualifications, dates of appointments and the like. Essentially a one-stop shop for the most accessible levels of government, localboards.org was smarty built to be sclable to various levels of government and a wide array of geographies.

It was the first-place project from Hack Omaha III, an open-data coding competition.

Problem identification:

Given a Microsoft Word file, our team was asked to build a system for displaying, distributing and managing details about boards within Nebraska focusing mainly on Douglas County.

Initial Solution Identified:

Build a RESTful API that supported 90% of the business logic as a solution. This would enable anyone to build a client around the web service and gain access to all of the functions that are envisioned.

Construct a user interface to demonstrate a subset of the features envisioned for the web service.

Obstacles with Data:

  • ALL of the content was in non-structured MS Word files.
  • No two documents shared the same format or level of detail.
  • Large gaps in the data presented to the public
  • Unable to locate board members
  • Unable to determine what all of the seats/positions a board may have (only vacancies were documented.)
  • Missing historical, current and future detail on initiatives and/or agendas.
  • Missing historical meeting minutes about decisions made by the boards.

Final Solution Developed:

A proof-of-concept web service was constructed that displayed all boards by state only. We incorporated the use of sub domains to shard information by county & state.

The solution was fast and provided a good roadmap on how to expand the web service to support any city, county and state in the country.

The user interface constructed displayed information for desktop & mobile devices.
